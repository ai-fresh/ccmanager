Getting Started with CC Gate

This guide walks you through installing CC Gate, setting up the PermissionRequest hook, and configuring auto-accept so you can start using it with your Claude Code sessions immediately.

System Requirements

Before installing CC Gate, make sure your system meets the following requirements:

  • ✓
    macOS 13.0 Ventura or later — CC Gate uses MenuBarExtra and other modern SwiftUI APIs that require Ventura or newer. macOS 14 Sonoma and 15 Sequoia are fully supported.
  • ✓
    Claude Code CLI installed — CC Gate is a companion app for Claude Code. You must have it installed and authenticated before CC Gate can do anything useful. Install with npm install -g @anthropic-ai/claude-code.
  • ✓
    At least one Claude Code project — CC Gate reads session data from ~/.claude/projects/. This directory is created automatically when you run Claude Code for the first time in any folder.
  • ✓
    ~/.claude/ directory — CC Gate needs read access to your Claude configuration directory. On sandboxed builds, you will be prompted to grant this access on first launch.

Installation

CC Gate is distributed as a signed and notarized macOS app. There are two ways to install it:

Option A — DMG (Recommended)

The DMG installer is the simplest way to install CC Gate:

  1. Go to the GitHub Releases page and download the latest CC Gate-x.y.z.dmg file.
  2. Open the downloaded DMG by double-clicking it.
  3. Drag CC Gate.app to your Applications folder in the window that opens.
  4. Eject the DMG volume.

Option B — PKG (System Installer)

The PKG installer places CC Gate in /Applications automatically and is useful for managed deployments:

  1. Download the latest CC Gate-x.y.z.pkg from GitHub Releases.
  2. Double-click the PKG and follow the installer prompts.
  3. Enter your administrator password when requested.

First Launch Security Bypass

Because CC Gate is distributed outside the Mac App Store, macOS Gatekeeper may block it on first launch even though it is properly signed and notarized. If you see a message saying the app "cannot be opened because the developer cannot be verified":

  1. Find CC Gate.app in your Applications folder in Finder.
  2. Right-click (or Control-click) the app icon.
  3. Choose Open from the context menu.
  4. In the dialog that appears, click Open again to confirm.

This only needs to be done once. After that, CC Gate will launch normally by double-clicking or from Spotlight.

Note: CC Gate is signed with an Apple Developer ID certificate and notarized by Apple. The security bypass above is only required because it is not distributed through the App Store. Gatekeeper still verifies the signature before you confirm the first launch.

First Run

When you launch CC Gate for the first time, a small terminal icon appears in your macOS menu bar — the narrow strip of icons at the top right of your screen. CC Gate has no Dock icon by design; it lives entirely in the menu bar.

What you see

Click the CC Gate icon in the menu bar to open the popup. The main popup shows:

  • Project list — every project directory where Claude Code has been used, detected automatically from ~/.claude/projects/.
  • Status indicators — colored dots and spinners next to each project name showing whether Claude Code is currently active, waiting for input, or idle. See Status Indicators for full details.
  • Session count — how many Claude Code sessions exist for each project.
  • Last activity time — when the most recent session file was modified.
  • Bell icon — opens the Auto-Accept Questions window where you control permission tiers.
  • Settings icon — opens the full Settings panel for CC Gate configuration.
Note: If you don't see any projects in the list, make sure Claude Code has been run at least once in a project directory. The ~/.claude/projects/ folder and its contents are created by Claude Code when it first runs.

Install the Hook

The hook is the most important part of CC Gate's auto-accept functionality. Without the hook, CC Gate can monitor sessions and show status indicators, but it cannot intercept permission requests or automatically approve tool usage.

What the hook does

Claude Code has a built-in mechanism called PermissionRequest hooks. Before Claude executes any tool that requires your approval — such as writing a file, running a terminal command, or making a web request — it calls this hook with the tool name and arguments. The hook can then return an allow or deny decision.

CC Gate installs a shell script at ~/.claude/hooks/ccmanager-permissions.sh and registers it in ~/.claude/settings.json. When a PermissionRequest fires, this script checks CC Gate's auto-accept configuration file at ~/.claude/.ccmanager/auto-accept-config.json in under a millisecond, and either approves the request immediately or routes it to the CC Gate UI for manual approval.

How to install

  1. Click the CC Gate icon in the menu bar.
  2. Click the Settings icon (gear icon) to open Settings.
  3. Click the Hook tab.
  4. Click Install Hook.
  5. The status badge should turn green and show Installed.
Important: The hook only applies to Claude Code sessions started after the hook is installed. Any existing Claude Code sessions (terminal windows already running claude) will not pick up the hook until you start a new session. Type /clear in Claude Code or open a new terminal window to start a fresh session.

Hook auto-upgrade

When CC Gate is updated and the hook script changes, CC Gate detects that the installed hook version is outdated on the next app startup and automatically reinstalls the new version. A notification banner appears in the popup letting you know this happened. You do not need to manually reinstall the hook after app updates.

Hook auto-repair

Claude Code sometimes overwrites ~/.claude/settings.json when you run certain commands, removing the hook registration. CC Gate detects this and automatically re-registers the hook on the next refresh cycle. The hook script file itself is not affected — only the registration entry in settings.json may need re-adding.

Grant Permissions

CC Gate needs Accessibility permission to send keyboard input to terminal windows when manually approving or denying questions from the CC Gate UI. Without this permission, the manual approval buttons in the Questions window may not function correctly.

How to grant Accessibility access

  1. Open CC Gate Settings.
  2. Click the Permissions tab.
  3. Click Grant Accessibility Access.
  4. macOS will prompt you to open System Settings > Privacy & Security > Accessibility.
  5. Find CC Gate in the list and enable the toggle next to it.
Tip: If you only plan to use the auto-accept feature and never manually answer questions from the CC Gate UI, Accessibility access is optional. It is only required for the manual Approve/Deny buttons in the Questions window to send keystrokes to the terminal.

Configure Auto-Accept

Auto-accept is where CC Gate really shines. Instead of switching to a terminal window to answer every "May I read this file?" or "May I run this command?" question, CC Gate can answer them automatically based on permission tiers you configure.

Opening the Questions window

  1. Click the CC Gate icon in the menu bar.
  2. Click the bell icon in the top-right corner of the popup.
  3. The Questions window opens, showing the 4-tier toggle controls.

Enabling your first tier

For a first-time setup, enabling the Read tier is a safe and sensible starting point:

  1. In the Questions window, find the Read row (shown in blue).
  2. Toggle the switch to enable it.
  3. CC Gate will now automatically approve Read, Glob, Grep, WebSearch, WebFetch, and LS tool requests without showing them to you.

The 4 tiers available are:

  • Read (blue) — Read-only operations. Minimal risk. Recommended for everyone.
  • Write (orange) — File creation and modification. Low risk if you use git, since changes are reversible.
  • Execute (red) — Bash and terminal commands. High risk. Only enable if you trust the codebase and Claude's judgment.
  • Plan (purple) — ExitPlanMode approval. Minimal risk. Allows Claude to proceed from planning to execution phase.
Tip: Start with only the Read tier enabled for a few sessions to get comfortable with how CC Gate works. Add the Write tier when you feel confident. For most workflows, Read + Write provides an excellent balance of automation and safety.
Warning: Enabling Execute (Bash) gives Claude Code full shell access to your system. Only enable this for projects where you have high trust in both the codebase and Claude's decisions. Consider enabling it per-session rather than permanently.
