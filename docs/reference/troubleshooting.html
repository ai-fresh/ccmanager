Hook Issues

Hook shows "Not installed"

Go to Settings → Hook → click the Install button. CC Gate will write the hook script to ~/.claude/hooks/ccmanager-permissions.sh and register it in ~/.claude/settings.json.

If installation fails, verify that the ~/.claude/ directory exists. If it does not, open Claude Code at least once to create it, then retry the installation.

Hook shows "Outdated"

CC Gate detected that the installed hook file version does not match the current expected version. Uninstall the hook (Settings → Hook → Uninstall), then reinstall (Install). The new hook takes effect for new Claude Code sessions.

Questions not appearing after installing hook

The hook only applies to new Claude Code sessions. Claude Code snapshots hook configuration when a session starts. Close your current Claude Code session and open a new one. If using /continue, exit fully and restart.

Claude Code overwrites settings.json

This can happen when Claude Code updates or resets its configuration. CC Gate detects this on the next refresh cycle and automatically re-adds the hook registration entry. If auto-repair does not trigger, go to Settings → Hook → Uninstall → Install to force a fresh registration.

Safe commands bypass the hook

Expected behavior

Commands like echo, ls, git status, and other read-only operations may not trigger a PermissionRequest event. Claude Code only fires PermissionRequest for operations it considers potentially sensitive. This is by Claude Code's design and is expected behavior — CC Gate cannot intercept operations that Claude Code does not send permission requests for.

Auto-Accept Issues

Questions not being auto-accepted

Open the Questions window (bell icon in the CC Gate menu) and check which tiers are enabled. The auto-accept tier must match the tool being requested. For example, if only Read tier is enabled, Bash commands (Execute tier) will still require manual approval.

Also confirm the hook is installed and the current Claude Code session started after the hook was installed.

Wrong tier level shown for a tool

Verify which tier a tool belongs to in the Auto-Accept documentation. Unknown tools (those not in CC Gate's tier map) always require manual approval regardless of which tiers are enabled. This is intentional for safety.

Auto-accept stopped working after app update

A CC Gate update may have changed the hook version. Check Settings → Hook — if it shows "Outdated", reinstall the hook. Then start a new Claude Code session so the updated hook is picked up.

Auto-accept tiers reset to Off after restart

This was a known bug fixed in v9.0 (the @Published didSet in init issue). If you are seeing this on v9.0 or later, ensure you are running the latest version. Download from GitHub Releases and reinstall.

Project Display Issues

Projects show "Unknown" status

Expected for old projects

Older Claude Code sessions do not include an originalPath field in their session data. CC Gate cannot resolve the path for these projects and shows "Unknown" status. This is expected for projects created before CC Gate started tracking them. The project will show correctly once a new Claude Code session is started for it.

Duplicate projects in the list

CC Gate deduplicates projects by originalPath. If you see apparent duplicates, they likely have different resolved paths (e.g. one via symlink and one via absolute path, or two separate checkouts in different directories). They are genuinely different entries.

A project is not showing in the list

The project must have at least one session file in ~/.claude/projects/. Open Claude Code in the project directory and send at least one message to create a session. CC Gate will pick it up on the next refresh cycle.

Project status appears outdated for an active session

CC Gate refreshes at the configured interval (default 15 seconds). If the status looks stale, wait for the next refresh, or increase the refresh frequency in Settings → General → Monitoring. The minimum is 5 seconds.

Git Worktree Issues

Worktree stuck after a failed merge or operation

Run git worktree prune in terminal from the main repository directory. This removes references to worktrees whose directories no longer exist. If the worktree directory is still present but broken, delete it manually: rm -rf /path/to/worktree, then run git worktree prune again.

Merge conflict in a worktree

Resolve conflicts manually in the worktree directory using your editor or git command line tools. After resolving all conflicts, run git add . and git commit in the worktree directory. Then return to CC Gate — it will detect the resolved state on the next refresh.

Push from worktree rejected

Pull the latest changes in the worktree: cd /path/to/worktree && git pull origin main (or the relevant branch). Resolve any merge conflicts, then retry the push from CC Gate.

Worktree not appearing in CC Gate

CC Gate reads worktrees via git worktree list for each tracked project. If a worktree does not appear, ensure it was created with git worktree add from within the project repository (not a copy of the repo). CC Gate will pick it up on the next refresh.

AI Commits Issues

"No session found"

A Claude Code session file must exist for the project in ~/.claude/projects/. Open the project in Claude Code and send at least one message. The session file is created on the first message. Then return to CC Gate's CommitSheet and try again.

Empty or generic commit message generated

This usually means the session context was empty (no messages since the last commit) or the diff was empty. Ensure you have staged or unstaged changes. Try adding a refinement note describing what you worked on — CC Gate will use it to generate a more specific message.

API key error

Go to Settings → AI → Test. If the test fails, the key may be invalid or have zero remaining credits. Regenerate your key at console.anthropic.com, paste the new key into Settings → AI, and test again.

Diff too large / chunking errors

Large diffs are automatically chunked. If errors persist even after chunking, the diff may be extremely large (over 500 KB). Add large generated files to .gitignore — common culprits include package-lock.json, yarn.lock, compiled output, database dumps, and binary assets. Use git diff --stat to identify the largest contributors.

GitHub Integration Issues

"Not connected" / token not recognized

Add or re-enter your GitHub Personal Access Token in Settings → GitHub. The token must have repo scope. After saving, the status badge should turn green. If it stays gray, the token may be malformed — copy it directly from the GitHub token page.

Push rejected by GitHub

The remote branch has commits that are not in your local branch. Run git pull origin main (or the relevant branch name) in terminal, resolve any conflicts, then retry the push from CC Gate's CommitSheet.

Fork operation failed — fork already exists

GitHub's API returns an error if a fork of the repository already exists under your account. Navigate directly to github.com/your-username/repo-name. CC Gate cannot overwrite or merge an existing fork.

Token expired

GitHub Personal Access Tokens with a set expiration date stop working after the expiration date. Generate a new token at github.com/settings/tokens, then update it in Settings → GitHub → paste new token → Save token.

App / Performance Issues

High CPU or battery usage

Increase the refresh interval in Settings → General → Monitoring. The default is 15 seconds. At 5 seconds, CC Gate runs frequent ps and file system scans which noticeably increase CPU usage. Try 30 seconds or 60 seconds if you are on battery.

App not launching / blocked by Gatekeeper

If macOS blocks CC Gate at launch, open System Settings → Privacy & Security. Scroll to the Security section — you should see a message about CC Gate being blocked. Click "Open Anyway". CC Gate is signed and notarized with a Developer ID certificate, so this prompt should appear only on first launch after download.

Menu bar icon missing

CC Gate runs exclusively as a menu bar app with no Dock icon. If the icon is not visible, the app may not be running. Open Finder → Applications → CC Gate.app and double-click to launch. If you see no icon after launch, check if menu bar icon hiding is enabled in macOS (System Settings → Control Centre → Menu Bar Only section).

Settings window does not open

Click the CC Gate menu bar icon, then click the gear icon in the popup. If the window opens behind other windows, check Mission Control or click the gear icon again. On macOS 15+, the settings window may open in the background — look for it in the Dock or use Command+Tab.

Debug Logs

CC Gate writes structured debug logs to a persistent file. Always include both logs when filing a bug report.

CC Gate application log

tail -f /tmp/ccmanager-debug.log

Hook log

tail -f ~/.claude/.ccmanager/hook.log

The app log uses prefixed tags to identify the source of each entry: [Scanner] for project scanning, [AutoAccept] for tier decisions, [ViewModel] for state changes, and [HookInstaller] for hook operations.

Filing a bug report Include your CC Gate version (Settings → About), macOS version, and both log files. Report issues at github.com/ai-fresh/ccgate/issues. Paste the relevant log lines rather than attaching the full file if possible, to make it easier to identify the relevant section.
