Hook System

The hook is CC Gate's connection point into Claude Code. It intercepts permission requests before they reach your terminal, enabling instant auto-approval of configured tool tiers and routing manual decisions to the CC Gate UI.

What is the Hook?

Claude Code has a built-in extensibility mechanism called hooks. Hooks are shell scripts that Claude Code calls at specific lifecycle events. CC Gate uses one specific hook type: PermissionRequest.

The PermissionRequest hook fires every time Claude Code is about to use a tool that requires your approval. This includes file operations (Read, Write, Edit), web requests (WebFetch, WebSearch), terminal commands (Bash), and any other tool that Claude Code treats as needing user consent.

Without CC Gate, Claude Code handles these itself — pausing and printing a permission prompt to your terminal, then waiting for you to type y or n. With CC Gate's hook installed, every permission request passes through the hook script first, giving CC Gate the opportunity to approve it automatically or display it in the Questions window for your review.

No hacks: The hook system is a first-party, officially supported Claude Code feature. CC Gate doesn't scrape terminal output, inject keystrokes, or use any unofficial APIs. The hook mechanism is documented by Anthropic and designed for exactly this kind of integration.

How It Works

Here is the exact sequence of events when Claude Code encounters a tool requiring permission:

⚡

Claude Code triggers PermissionRequest

Claude Code wants to use a tool (e.g., Bash to run a command). Before proceeding, it invokes the registered PermissionRequest hook with the tool name and arguments as environment variables.

↓
📄

Hook script reads config file

The hook script at ~/.claude/hooks/ccmanager-permissions.sh reads ~/.claude/.ccmanager/auto-accept-config.json directly from disk. This is a simple JSON file read — no network calls, no IPC, no waiting. It takes under 1ms.

↓
❓

Check: is this tool's tier enabled?

The hook looks up the requested tool name in its internal tier mapping. If the tool belongs to Read, Write, Execute, or Plan tier and that tier is enabled in the config file, the decision is allow.

↓
✓

Auto-accept: output allow decision

If the tier is enabled, the hook immediately outputs the JSON decision: {"hookSpecificOutput":{"hookEventName":"PermissionRequest","decision":{"behavior":"allow"}}}. Claude Code receives this and proceeds without any user interaction.

↓ (if not auto-accepted)
🔔

Manual review: write question file

If the tier is disabled or the tool is unknown, the hook writes a question JSON file to ~/.claude/.ccmanager/questions/. CC Gate's QuestionWatcher detects this new file within 2 seconds and displays the question in the Questions window for your manual approval or denial.

↓
📝

Answer written, hook returns

When you click Approve or Deny in the CC Gate Questions window, CC Gate writes the answer to ~/.claude/.ccmanager/answers/. The hook is polling for this answer file. When it appears, the hook reads it and outputs the corresponding allow or deny decision to Claude Code.

Key files involved

📄
~/.claude/hooks/ccmanager-permissions.sh

The hook script itself. Generated by CC Gate's HookInstaller. Contains a version comment like # CCManager Permission Hook v9.0.0. Do not edit manually.

📄
~/.claude/settings.json

Claude Code's global settings file. CC Gate registers the hook here under hooks.PermissionRequest with an empty matcher (matches all tools) and a 3600-second timeout.

📄
~/.claude/.ccmanager/auto-accept-config.json

The auto-accept configuration file read by the hook. Updated by CC Gate every time you toggle a tier. Format: {"read": true, "write": false, "execute": false, "plan": false}

📄
~/.claude/.ccmanager/hook.log

The hook's own log file. Contains one line per permission request showing the tool name, tier lookup result, and decision made. Useful for debugging unexpected behavior.

📁
~/.claude/.ccmanager/questions/

Directory where the hook writes question JSON files for tools that need manual approval. CC Gate polls this directory every 2 seconds.

📁
~/.claude/.ccmanager/answers/

Directory where CC Gate writes answer JSON files after you click Approve or Deny. The hook polls for answer files matching the question ID.

Installing the Hook

The hook is installed from Settings in CC Gate:

  1. Click the CC Gate icon in the menu bar.
  2. Open Settings (gear icon).
  3. Click the Hook tab.
  4. Click Install Hook.

After installation, the status badge on the Hook tab changes to ● Installed. The Hook tab also shows the current hook version and the path where it was installed.

CC Gate performs the following steps during installation:

  1. Generates the hook script content from the current template in HookInstaller.generateHookScript().
  2. Writes the script to ~/.claude/hooks/ccmanager-permissions.sh.
  3. Sets the file permissions to 755 (executable).
  4. Reads ~/.claude/settings.json, adds or updates the hooks.PermissionRequest entry, and writes it back.
  5. Writes the initial auto-accept-config.json file reflecting your current tier settings.

Auto-Repair

Claude Code occasionally overwrites ~/.claude/settings.json during certain operations — for example, when you run /doctor or when Claude Code performs a self-update. This can remove the hook registration entry, causing the hook to stop being called even though the script file still exists on disk.

CC Gate detects this situation automatically. On each refresh cycle (every 15 seconds by default), CC Gate reads ~/.claude/settings.json and checks whether the hook registration is present. If the hook script exists but the registration is missing, CC Gate silently re-adds the registration entry and saves the file.

You do not need to do anything — the auto-repair happens transparently in the background. The Hook tab status badge continues showing ● Installed correctly.

Note: Auto-repair only fixes the settings.json registration. If the hook script file itself is deleted (e.g., someone ran rm ~/.claude/hooks/ccmanager-permissions.sh), auto-repair does not recreate it. In that case, open Settings → Hook and click Install Hook to reinstall.

Auto-Upgrade

When CC Gate is updated to a new version that includes hook script changes, the installed hook may be outdated. An outdated hook can miss new tier mappings, have bugs that were fixed, or lack new features.

On each app startup, CC Gate reads the version comment from the installed hook script and compares it to the current expected version. If they differ, CC Gate automatically reinstalls the hook with the new script, showing a brief notification banner in the CC Gate popup to let you know the upgrade happened.

The upgrade process is identical to a fresh install — it overwrites the script file and verifies the settings.json registration. The process takes less than a second and requires no user interaction.

Important: After a hook upgrade, any existing Claude Code sessions will continue using the old hook version until they are restarted. The hook script is read once per Claude Code session at session startup — mid-session hook changes are only picked up on a new session. Open a new terminal window and start a fresh claude session to use the upgraded hook immediately.

Uninstalling the Hook

To remove the CC Gate hook:

  1. Open Settings → Hook tab.
  2. Click Uninstall Hook.

Uninstalling removes the hook script file from ~/.claude/hooks/ and removes the PermissionRequest registration from ~/.claude/settings.json. After uninstalling, Claude Code falls back to its built-in terminal prompt for all permission requests.

The auto-accept config file and question/answer directories are left in place — they are not removed during uninstall. This means your tier settings are preserved if you reinstall the hook later.

Important Notes

  • Hook applies only to new sessions. Existing Claude Code terminal sessions that were started before hook installation do not pick up the hook. You must start a new claude session (new terminal window, or type exit and relaunch) to activate the hook.
  • Safe commands may bypass the hook. Claude Code has its own internal judgment about which commands are low-risk enough to execute without a permission prompt. Simple read-only commands like echo, pwd, ls, and git status may not trigger PermissionRequest at all, meaning CC Gate never sees them. This is expected and intentional — Claude Code is being conservative about what it considers permission-worthy.
  • Hook timeout is 3600 seconds. The hook registration specifies a 3600-second (1 hour) timeout for each PermissionRequest. This is intentionally long to ensure that manual approval prompts in the CC Gate Questions window don't time out while you step away from your computer. Claude Code will wait up to 1 hour for a manual decision.
  • AskUserQuestion is never intercepted. The hook intentionally passes through AskUserQuestion events. These are genuine questions where Claude needs your input, not just permission to use a tool. They are handled by Claude Code's native terminal prompt.

Troubleshooting

Hook tab shows "Not Installed" after app update

If the hook version changed in a CC Gate update, the Hook tab may briefly show the hook as outdated or not installed before auto-upgrade runs. Wait a few seconds for CC Gate to complete startup, then check again. If the problem persists, manually click Install Hook.

Questions are not appearing in the CC Gate window

Check the following in order:

  1. Confirm the hook is installed — Hook tab should show ● Installed.
  2. Confirm you started a new Claude Code session after installing the hook. Existing sessions don't pick up the hook.
  3. Check ~/.claude/.ccmanager/hook.log — if the hook is being called, you'll see log lines appearing when Claude Code requests permissions.
  4. If the log shows no activity, verify that ~/.claude/settings.json contains the PermissionRequest hook registration.

Hook shows "outdated" error message

This means the installed hook script is an older version. CC Gate should auto-upgrade on startup, but if it doesn't, open Settings → Hook and click Uninstall Hook, then Install Hook to force a fresh install.

Auto-accept is not working even with tiers enabled

Verify the auto-accept config file exists and has the correct content:

cat ~/.claude/.ccmanager/auto-accept-config.json

The output should be a JSON object with boolean values for each tier. If the file is missing, toggle any tier in the CC Gate Questions window to force it to be written. If the file has unexpected content, try toggling tiers off and back on.

Viewing the hook log

The hook writes a log entry for every permission request it processes. This is the most reliable way to verify the hook is running:

tail -f ~/.claude/.ccmanager/hook.log

Run this in a terminal, then trigger a permission-requiring action in Claude Code. You should see a new log line within a second or two.
