Background Test Runner

Automatically run tests when files change and inject results into Claude Code's context so Claude always knows the current test status.

Overview

CC Gate's Test Runner watches your project files for changes and automatically runs tests in the background. Results are injected into Claude Code's context so Claude always knows the current test status before suggesting any changes. This closes the feedback loop: Claude writes code, CC Gate runs tests, Claude sees results and adjusts — without you pasting error messages manually.

Developer Mode Feature The Test Runner is hidden behind Developer Mode. Enable it by tapping the version number 7 times in Settings → About, or via defaults write. See "Enabling Developer Mode" below.

Enabling Developer Mode

Developer Mode unlocks experimental and power-user features that are not shown in the standard UI. The Test Runner is one of these features.

  1. Open Settings → About

    Click the CC Gate menu bar icon → Settings → About (the Info tab at the bottom of the settings window).

  2. Tap the version number 7 times

    Click the version number label (e.g. "CC Gate 2.11.12") rapidly 7 times. After the 7th tap, a "Developer Mode" label appears with a checkmark indicator.

  3. Alternative: Terminal command

    If you prefer, run this in Terminal and restart CC Gate:

    defaults write com.adrianmagiera.CCManager developerModeEnabled -bool YES

Once Developer Mode is active, a "Developer" section appears in Settings with the Test Watcher toggle and additional configuration options.

Test Watcher

With Developer Mode enabled, go to Settings → Developer and toggle "Test Watcher" on. CC Gate immediately begins watching the active project's source directory for file changes.

What triggers a test run

  • Any source file change detected by the filesystem watcher: .swift, .py, .js, .ts, .cs files
  • A manual "Run Tests" trigger from the CC Gate UI (project card or Test Results window)

Supported test frameworks

Test Results in Claude Code Context

After each test run, CC Gate writes the results to a JSON file. A secondary hook script reads this file and prepends the test status to Claude Code's context at the start of each message. Claude sees whether tests pass, which tests fail, and the error output — before deciding what changes to make next.

This means Claude can proactively say "the tests are currently failing with X error — let me fix that before adding the new feature" without you having to paste anything.

Tip: Copy failures to Claude The Test Results window has a "Copy to Claude" button that copies the full failure output formatted for pasting into a Claude Code message. Useful when you want Claude to focus on a specific failure without waiting for the next automatic context injection.

Hook Architecture

The Test Runner uses a second hook registration separate from CC Gate's main permission hook. This keeps concerns cleanly separated and allows independent enable/disable.

ccmanager-context.sh Always active. Handles PermissionRequest events and pending context injection.
ccmanager-tests.sh Test results only. Dynamically registered/unregistered when Test Watcher is toggled.

When you disable the Test Watcher, CC Gate removes the ccmanager-tests.sh entry from ~/.claude/settings.json entirely. Claude Code no longer receives test context for that session. Enabling the watcher re-registers it.

New sessions required Claude Code snapshots hook registrations at the start of each session. If you enable or disable the Test Watcher, the change takes effect for new Claude Code sessions only. Close and reopen Claude Code for the change to apply.

Viewing Results

CC Gate shows test results in a dedicated TestResultsWindowView, accessible from the project card or via the menu bar icon context menu when tests are running.

  • Pass / fail status — summary badge showing total passed and failed counts
  • Individual test cases — expandable list of each test with pass/fail indicator
  • Error output — full stderr and stdout for failing tests
  • Copy to Claude — formats and copies failure details for pasting directly into Claude Code
  • Run again — manually triggers a fresh test run without waiting for a file change

Troubleshooting

Tests not running

Ensure Developer Mode is enabled (version label shows "Developer Mode" checkmark in Settings → About) and that the Test Watcher toggle is on in Settings → Developer. If both are set correctly, check that the project has source files with supported extensions in its directory.

Wrong test command being run

CC Gate auto-detects the command but may guess incorrectly for non-standard project layouts. Go to Settings → Developer → Test command and enter the exact shell command you would run in terminal (e.g. ./vendor/bin/phpunit or cargo test).

Results not appearing in Claude Code

Restart your Claude Code session. Hook changes only apply to new sessions — the test hook registration (or removal) takes effect when Claude Code starts a new conversation, not mid-session.

Stale test results after disabling watcher

When you toggle the Test Watcher off, CC Gate automatically deletes the test-results.json file so stale results are not injected into future Claude Code sessions. If you see old results, toggle the watcher off and back on, or delete the file manually at ~/.claude/.ccmanager/test-results.json.
