Status Indicators

CC Gate shows live status for every Claude Code project in the main popup. This reference explains what each status state means, how it is determined from session files, and how to configure the refresh rate for your workflow.

Overview

The CC Gate popup provides at-a-glance visibility into all your Claude Code projects without needing to switch to any terminal window. Each project card shows a status indicator reflecting the current state of Claude Code sessions in that directory.

Status is determined by reading and parsing the session files that Claude Code writes to ~/.claude/projects/. CC Gate does this on a configurable timer, refreshing status for all projects at once. No background processes are monitored and no system calls like ps or lsof are used — status is inferred entirely from filesystem timestamps and session file content.

Status States

There are four possible states for a project. Each has a distinct visual appearance:

Active

Green pulsing dot — Claude Code is currently running a task

The Active state means a Claude Code process is running in this project directory and is actively doing work. The dot pulses with a subtle glow animation to draw your attention. You'll see this state while Claude is thinking, planning, or executing tool calls. A project may stay in Active state for extended periods during long-running tasks like large refactors, test runs, or multi-file edits.

Waiting

Orange dot — Claude Code is waiting for your input or permission

The Waiting state means Claude Code has paused and is expecting something from you. This includes two scenarios: a tool permission request waiting for your approval (which CC Gate can handle automatically with Auto-Accept), or a direct question from Claude that requires your input. The orange color is intentionally attention-grabbing — something needs your response before Claude can continue.

Idle

Gray dot — No active Claude Code process

The Idle state means there is no running Claude Code process for this project. The project has session history (it appears in the list), but Claude Code is not currently running in that directory. This is the normal resting state of any project between sessions. You'll see all your projects in Idle state when Claude Code is not open in any terminal.

Outdated

Faded orange dot — Session data is stale (older than 24 hours)

The Outdated state appears when the most recent session file for a project was last modified more than 24 hours ago. This usually means you worked on the project yesterday or earlier but haven't opened Claude Code in it since. It's informational — not an error. The outdated indicator helps you distinguish freshly-used projects from ones that haven't been touched in a while.

How Status is Determined

CC Gate does not directly monitor running processes. Instead, it reads the session files that Claude Code maintains in ~/.claude/projects/ to infer the current state. Here is how each status is determined:

Project discovery

CC Gate's ProjectScanner reads ~/.claude/projects/ to find all projects. Each subdirectory in this folder corresponds to one project Claude Code has been used in. The folder name is an encoded form of the project's absolute path (encoded by Claude Code itself). CC Gate reads the originalPath field from session files to decode the actual path.

Session file parsing

Each project directory contains .jsonl (JSON Lines) session files. Claude Code appends one JSON object per line to these files as it works. Each line describes a message or event in the session. CC Gate's SessionStatusChecker reads the last few lines of the most recent .jsonl file for each project to determine:

  • Last message type — if the last message was an assistant message still in progress, the session is Active. If it was a permission request, it's Waiting. If it was a completion, it's Idle.
  • File timestamp — how recently the session file was modified. If the last modification was more than 24 hours ago, the state becomes Outdated.
  • Active process detection — CC Gate compares the modification time of the session file against the current time. A file modified within the last few seconds (adjusted for the refresh interval) indicates an actively running session.

Caching

Session status is cached for 4 seconds by the SessionStatusChecker. This means rapid successive reads — such as when the popup is opened while a refresh is in progress — return the cached status rather than re-reading all session files from disk. The cache expires automatically, so the status shown is never more than 4 seconds stale within a refresh cycle.

Refresh Rate

CC Gate refreshes status on a configurable timer. The default is 15 seconds, chosen as a balance between freshness and energy efficiency. You can adjust this in Settings → General → Monitoring using the "Częstotliwość odświeżania" (Refresh frequency) slider.

Interval Energy Impact Status Freshness Best For
5 seconds High CPU usage Very fresh (≤5s delay) Active monitoring, plugged in
15 seconds (default) Balanced Fresh (≤15s delay) Most workflows
30 seconds Low energy Acceptable (≤30s delay) Battery saving, background use
60 seconds Minimal impact Coarse (≤60s delay) Always-on overnight use

The high energy impact at 5 seconds comes from the system calls involved in scanning session files. CC Gate uses FileManager APIs to list and read files in ~/.claude/projects/, which triggers filesystem activity that macOS tracks as CPU usage. At 15 seconds, the impact is negligible in Activity Monitor.

Tip: If you mostly work with one or two projects at a time and run CC Gate continuously in the background, consider using 30 or 60 seconds. The status will still update reliably — you'll just see it a bit later after changes occur.

Dynamic caching (v2.6.5+)

CC Gate uses a dynamic caching system that adapts to your refresh interval:

  • ClaudeProcessDetector cache — expires at refreshInterval - 1 second. At 15s refresh, cache expires after 14s.
  • QuestionWatcher polling — checks for new question files every 2 seconds, independently of the main refresh timer.
  • SessionStatusChecker cache — 4 seconds, shared across all concurrent reads within a refresh cycle.

This means that even at a 60-second refresh interval, new permission questions appearing in ~/.claude/.ccmanager/questions/ are still picked up within 2 seconds by the QuestionWatcher. The slower refresh only affects project status updates in the popup, not the responsiveness of the Questions window.

Menu Bar Icon

The CC Gate icon in the macOS menu bar also reflects the overall state of all your projects:

📄 CC Gate
📄 CC Gate 2
📄 CC Gate !
  • Default icon (no badge) — All projects are Idle or Outdated. No Claude Code sessions are active.
  • Green badge with count — One or more projects have Active sessions. The number shows how many projects are currently active.
  • Orange badge with exclamation — One or more projects are in Waiting state, meaning Claude Code is paused and needs your attention. This badge has higher priority than the Active badge.

The badge makes it easy to notice when Claude Code has stopped and is waiting for your approval — even when you're in a different app. You don't need to keep checking the popup; the menu bar icon alerts you when action is needed.

Note: If you use the Auto-Accept feature with appropriate tiers enabled, the Waiting state should be rare in practice — most permission requests are handled automatically before reaching you. You'll primarily see the orange badge for AskUserQuestion prompts and unknown tools that require manual approval.
