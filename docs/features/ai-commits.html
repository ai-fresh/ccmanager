AI Commit Messages

Generate semantically accurate Conventional Commit messages from your Claude Code session context and git diff — no manual description required.

Overview

CC Gate's AI Commit Messages feature analyzes your Claude Code session context and git diff to generate semantically accurate commit messages — without you having to describe your changes manually. Instead of writing "fixed some stuff" or trying to recall what you built over the last hour, CC Gate reads what Claude Code actually did and produces a precise, structured commit message in seconds.

Pro Feature AI Commit Messages requires a CC Gate Pro plan and an Anthropic API key. See Settings → Pro to upgrade, and Settings → AI to enter your key.

Requirements

  • Pro plan — AI Commits is a Pro feature (Settings → Pro)
  • Anthropic API key — Settings → AI → enter your key from console.anthropic.com
  • Claude Code session — an active or recent .jsonl session file for the project
  • GitHub token (optional) — needed for the push workflow after committing

How It Works

When you open CommitSheet for a project, CC Gate kicks off a multi-step pipeline to build the most accurate commit message possible with the fewest tokens.

  1. Opening CommitSheet

    Right-click a worktree and choose "Commit changes", or click the commit button in a project card. CommitSheet opens with the staged diff loaded and generation begins automatically.

  2. DELTA Logic — only new messages since last commit

    CC Gate reads the timestamp of your last git commit, then filters the .jsonl session file to include only Claude Code messages that occurred after that commit. This avoids re-analyzing old context, saves tokens, and keeps the generated message focused on what actually changed in this work session.

  3. Session Context Extraction

    The filtered session messages are parsed: assistant messages (what Claude did) and user messages (what you asked) are combined into a structured conversation summary. Tool calls like Edit, Write, and Bash are extracted to identify which files were touched and what operations were performed.

  4. Fallback for Projects Without Sessions

    If no active session file is found, CC Gate looks for a CLAUDE.md or README.md in the project root and uses that as project context instead. The diff alone is used to generate the message.

  5. Large Diff Chunking

    Diffs exceeding 20,000 tokens are automatically split into chunks. Each chunk is summarized individually using Claude Haiku (fast and cheap), then all summaries are combined and passed to Claude Sonnet for the final commit message. This keeps costs low even on large refactors.

  6. Refinement Loop

    After the message is generated, you can edit it directly in CommitSheet. You can also add a short note — for example, "I was working on fixing the auth timeout" — and CC Gate will refine the message using that hint without regenerating from scratch. This is much faster and cheaper than a full regeneration.

Conventional Commits Format

CommitSheet uses structured Conventional Commits format. Every generated message follows this structure:

type(scope): short imperative description

Optional longer body explaining the why and how.
Can span multiple lines, 72 chars wide.
Field Options Description
Type feat, fix, docs, refactor, test, chore, style, perf, ci Nature of the change
Scope optional — e.g. git, hook, ui, auth Which part of the codebase
Description max ~72 chars, imperative mood Short summary of the change
Body optional multi-line Detailed explanation of why and how
Tip: Refine without regenerating Add a note in the refinement field (e.g. "focus on the performance improvements") and click Refine. This costs a fraction of a full regeneration and usually produces a better result than starting over.

Troubleshooting

"No session found"

Make sure Claude Code has processed at least one message for this project. The session file must exist in ~/.claude/projects/. Open the project in Claude Code and send a message, then try again.

Empty diff

CommitSheet detected no staged or unstaged changes. Make sure you have modified files before opening CommitSheet. Run git status in terminal to verify.

API key error

Verify your Anthropic API key in Settings → AI → Test button. If the test fails, the key may be invalid, revoked, or have insufficient credits. Regenerate at console.anthropic.com.

Token limit / chunking failure

Large diffs are automatically chunked. If you see errors even with chunking enabled, the diff may be extremely large (over 500 KB). Consider using .gitignore to exclude large generated files (e.g. package-lock.json, compiled assets, database files) from tracking.
