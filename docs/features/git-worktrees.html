Git Worktrees

Git worktrees let you check out multiple branches of the same repository into separate directories simultaneously. CC Gate makes creating, managing, and working across these worktrees seamless directly from your menu bar.

What are Git Worktrees?

A Git worktree is an additional working directory linked to the same repository. Unlike cloning the repository a second time, worktrees share the same .git folder — so they see the same branches, history, and remote configuration, but each has its own checked-out branch and working tree state.

The primary use case with CC Gate is running parallel Claude Code sessions: one Claude instance working on a feature branch while another maintains the main branch, without either session interfering with the other's working directory.

~/projects/my-app/ ← main [a1b2c3d]
~/projects/my-app-worktrees/feature-auth/ ← feature/auth [e4f5g6h]
~/projects/my-app-worktrees/fix-bug-42/ ← fix/bug-42 [i7j8k9l]

Each directory in the diagram above is a fully functional checkout. You can open a terminal in any of them, run claude, and have independent Claude Code sessions working simultaneously. CC Gate shows all three in the popup as separate worktree rows under the project card.

Git requirement: Worktrees require Git 2.5 or later. Run git --version to check. macOS ships with git, but you can also install a newer version via Homebrew with brew install git.

Viewing Worktrees

In the CC Gate main popup, each project card shows a disclosure arrow (>) on the right side if the project is a git repository with worktrees. Click the arrow to expand the worktree section.

Each worktree row in the expanded view shows:

  • Branch name — the currently checked-out branch in that worktree directory.
  • Commit hash prefix — the short hash of the HEAD commit (e.g., a1b2c3d), useful for identifying what state each worktree is in at a glance.
  • Status dot — the same colored status indicator used for the project overall, but scoped to Claude Code sessions running inside that specific worktree directory.
  • Dirty indicator — a badge appears if the worktree has uncommitted changes (modified or untracked files).

The main project card row always represents the primary worktree (the original repository checkout, usually on main or master). All additional worktrees are listed below it in the expanded section.

Creating a Worktree

To create a new worktree from CC Gate:

  1. In the main popup, expand a project card by clicking the > arrow.
  2. Click the + button in the worktree section header.
  3. The Create Worktree sheet opens.
  4. Enter a branch name for the new worktree (e.g., feature/my-new-feature).
  5. Select the base branch from the dropdown — this is what the new branch will be based on. Defaults to main.
  6. The worktree path is calculated automatically based on your repository location and branch name. You can optionally customize it.
  7. Click Create.

CC Gate runs git worktree add -b <branch> <path> <base> in the background. The new worktree directory is created immediately. A success notification confirms the location.

Tip: Use a consistent naming convention for your worktrees. A good pattern is feature/description, fix/issue-number, or experiment/name. The branch name directly determines the worktree directory name, so keep it concise and filesystem-friendly (no spaces).

Worktree Context Menu

Right-clicking any worktree row in the expanded project card opens a context menu with actions for that specific worktree:

🔗 Open in Cursor

✓ Commit Changes...

Open in Editor / Terminal

The editor and terminal options open the worktree directory directly in the selected application. CC Gate detects which editors and terminals you have installed and only shows available options. Supported editors: VS Code, Cursor. Supported terminals: Warp, iTerm2, Terminal.app.

Show in Finder / Copy Path

Show in Finder opens a Finder window with the worktree directory selected. Copy Path copies the absolute filesystem path to your clipboard — useful for cd commands or sharing the location with a team member.

Committing Changes

When a worktree has uncommitted changes (shown by the dirty indicator), you can commit directly from CC Gate without opening a terminal:

  1. Right-click the worktree row and choose Commit Changes...
  2. The Commit Sheet opens, showing a summary of changed files and a diff preview.
  3. Enter a commit message in the text field. If you have AI Commit Messages configured, click Generate to create a conventional commit message from the diff automatically.
  4. Review the auto-generated or manually written message and edit as needed.
  5. Click Commit.

CC Gate runs git add -A && git commit -m "<message>" in the worktree directory. The commit hash and success confirmation appear in the sheet after the operation completes.

Conventional commits: The commit message field supports the conventional commits format — type(scope): description. The AI commit message generator always produces conventional commit format messages. Examples: feat(auth): add OAuth2 login flow, fix(api): handle rate limit errors gracefully.

Merging to Main

When your work in a worktree is ready to be integrated back into the main branch:

  1. Make sure all changes in the worktree are committed (no dirty indicator).
  2. Right-click the worktree row and choose Merge to main...
  3. The Merge Sheet opens, showing the commits that will be merged and the target branch (defaulting to main).
  4. Confirm the merge options and click Merge.

CC Gate switches to the main branch, runs git merge <branch>, and switches back. The merge result — fast-forward, merge commit, or conflict notification — is displayed in the sheet.

Merge conflicts: If the merge produces conflicts, CC Gate will notify you and abort the merge. You must resolve conflicts manually in a terminal or editor. Open the main worktree in your preferred editor via the context menu, resolve the conflicts, and run git merge --continue in the terminal.

Deleting a Worktree

To remove a worktree that is no longer needed:

  1. Right-click the worktree row and choose Delete Worktree...
  2. A confirmation dialog appears, warning that the working directory and any uncommitted changes will be permanently lost.
  3. Confirm the deletion.

CC Gate runs git worktree remove --force <path>, which removes the working directory and deregisters the worktree from git. The branch itself is not deleted — it remains in the repository and can be checked out again later.

Warning: Deleting a worktree is irreversible if you have uncommitted changes. Always commit or stash your work before deleting. The confirmation dialog shows a list of uncommitted files if any exist, giving you a final chance to save your work before proceeding.

Git Health Checker

CC Gate includes a Git Health diagnostic tool (introduced in v2.11.12) that can identify and help remediate common git repository issues that might prevent normal worktree operations.

What it checks

🔎

Stuck merge state

Detects in-progress merges that were never completed or aborted, which block new merges from starting.

🔎

Broken refs

Finds corrupt or missing branch references that can cause git operations to fail unexpectedly.

🔎

Detached worktrees

Identifies worktree registrations pointing to directories that no longer exist on disk.

🔎

Lock files

Detects stale .git/index.lock files left behind by crashed git processes that block all operations.

Running the health check

The health check can be triggered from the context menu of any project card in the CC Gate popup. When issues are found, CC Gate presents each one with a description of what it means and a one-click Fix button that applies the remediation automatically (e.g., removing a stale lock file or running git worktree prune).

Troubleshooting

Merge conflicts must be resolved manually

CC Gate cannot resolve merge conflicts on your behalf. When a merge fails due to conflicts, open the project in your editor, look for conflict markers (<<<<<<<, =======, >>>>>>>), resolve them, stage the resolved files, and run git merge --continue or git commit.

Worktree shows as "stuck" or doesn't appear

If a worktree directory was deleted manually (outside CC Gate) without properly removing it from git, the worktree will be listed by git but have no corresponding directory. Run the Git Health Checker to detect and prune these orphaned registrations, or run git worktree prune in a terminal.

Push rejected after creating worktree

When pushing a new branch to a remote for the first time, git requires you to set the upstream tracking branch. Open the worktree directory in a terminal and run:

git push --set-upstream origin <branch-name>

If you have the GitHub integration configured, CC Gate can handle this automatically when you commit and push from the Commit Sheet.

Cannot delete worktree — "is checked out"

Git prevents deleting a worktree if another process has the directory locked or if the branch is currently checked out in another worktree. Make sure no terminal sessions are running in the worktree directory and try again. If the problem persists, close any editors pointing to that directory and retry.
