Advanced Auto-Accept

Go beyond Claude Code's built-in settings with 4 independent permission tiers.

Note: Claude Code now includes native auto-accept functionality. CC Gate extends this with a more granular 4-tier system — letting you allow reads but block writes, or allow writes but require manual approval for shell commands. Each tier is toggled independently from a native macOS UI.
Tier Tools Risk
Read Read, Glob, Grep, WebSearch, WebFetch Minimal — read-only
Write Edit, Write, MultiEdit, NotebookEdit Low — reversible with git
Execute Bash (all terminal commands) High — full shell access
Plan ExitPlanMode Minimal — approves plans

FAQ

How is CC Gate different from Claude Code's built-in features?

Claude Code is a command-line tool — it runs in a terminal window per project. CC Gate sits in your menu bar and gives you a unified view across all your projects at once. It also adds Git worktree management, multi-editor integration, and a more granular auto-accept system on top of what Claude Code provides natively.

Does CC Gate work with multiple projects?

Yes. CC Gate monitors all sessions in ~/.claude/projects/ and displays them in a single list with per-project status, activity time, and worktree information.

Do I need to use the auto-accept feature?

No. You can keep all auto-accept tiers disabled and use CC Gate purely as a project dashboard, session monitor, and question management UI.

What happens if CC Gate is closed?

Claude Code falls back to its default behavior — asking in the terminal. CC Gate is purely additive; closing it changes nothing about how Claude Code works.

What data does CC Gate collect?

CC Gate reads local session data from ~/.claude/projects/. No data is sent to external servers. Everything stays on your Mac.
